# Lab4 — Survey Mobile Application

## Опис проєкту

Цей застосунок реалізує просту **мобільну форму опитування користувача** для операційної системи Android.

Користувач вводить відповіді на запитання у текстові поля, після чого натискає кнопку **“Зберегти відповіді”**. Введені дані зберігаються у текстовий файл у внутрішній пам’яті застосунку.

Метою лабораторної роботи є ознайомлення з принципами створення мобільних застосунків для Android та реалізація застосунку **“Форма опитування”**.

## Функціональність

Застосунок дозволяє:

- відображати форму опитування;
- вводити відповіді у текстові поля;
- вводити ім’я користувача;
- вводити вік користувача;
- відповідати на кілька запитань;
- перевіряти заповнення основних полів;
- зберігати введені відповіді;
- записувати результати у текстовий файл;
- відображати повідомлення про успішне збереження;
- очищати поля після збереження відповідей.

## Технології

- Kotlin;
- Android Studio;
- Android SDK;
- Jetpack Compose;
- Material 3;
- Gradle;
- File API для запису даних у файл.

## Приклад роботи

1. Користувач запускає мобільний застосунок.
2. На екрані відкривається форма опитування.
3. Користувач вводить ім’я, вік та відповіді на запитання.
4. Натискає кнопку **“Зберегти відповіді”**.
5. Застосунок перевіряє, чи заповнені основні поля.
6. Якщо дані введені коректно, відповіді записуються у файл.
7. Після збереження користувач бачить повідомлення з шляхом до файлу.

## Результат

Програма створює або дописує текстовий файл:

```text
survey_answers.txt
```

Файл зберігається у внутрішній пам’яті застосунку за шляхом:

```text
/data/user/0/com.example.surveyapp/files/survey_answers.txt
```

У файл записуються:

- дата та час заповнення форми;
- ім’я користувача;
- вік користувача;
- відповіді на питання;
- коментар або побажання користувача.

## Структура основного файлу

Основна логіка застосунку розміщена у файлі:

```text
app/src/main/java/com/example/surveyapp/MainActivity.kt
```

У цьому файлі реалізовано:

- запуск головної активності;
- побудову інтерфейсу користувача;
- створення текстових полів;
- обробку натискання кнопки;
- перевірку заповнення форми;
- формування тексту для збереження;
- запис відповідей у файл.

## Основні елементи інтерфейсу

У застосунку використано такі елементи:

- `Text` — для відображення заголовків і підписів;
- `OutlinedTextField` — для введення відповідей;
- `Button` — для збереження результатів;
- `Column` — для вертикального розміщення елементів;
- `Scaffold` — для побудови основної структури екрана;
- `Toast` — для показу коротких повідомлень користувачу.
